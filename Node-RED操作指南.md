# Node-RED操作指南

## 一、安装Node-RED

### Windows系统安装

1. **安装Node.js**
   - 访问 https://nodejs.org/
   - 下载LTS版本
   - 安装Node.js

2. **安装Node-RED**
   - 打开命令提示符（CMD）
   - 输入：`npm install -g --unsafe-perm node-red`
   - 等待安装完成

3. **启动Node-RED**
   - 在命令提示符中输入：`node-red`
   - 等待启动完成
   - 浏览器访问：http://localhost:1880

## 二、安装Dashboard插件

1. **安装node-red-dashboard**
   - 在Node-RED目录下打开命令提示符
   - 输入：`npm install node-red-dashboard`
   - 重启Node-RED

2. **验证安装**
   - 访问 http://localhost:1880
   - 左侧面板应该有dashboard节点

## 三、导入扩展版流程（极坐标雷达图 + 记录浏览）

### 导入 nodered_flow_extended.json

1. **打开Node-RED编辑器**（http://localhost:1880）
2. **删除旧流程**（如果有）
   - 全选节点（Ctrl+A）→ 按 Delete
3. **导入新流程**
   - 点击右上角菜单（三条横线）→ `Import` → `Clipboard`
   - 打开 `nodered_flow_extended.json`，全选复制内容
   - 粘贴到文本框，点击 `Import`
4. **配置串口**
   - 双击任意 `serial` 节点（粉色）
   - 在 `Serial Port` 下拉选择你的 COM 口
   - 确保 Baud rate 为 `115200`
   - 点击 `Done`，然后点 `Deploy`

## 四、查看Dashboard

1. **访问Dashboard**
   - 浏览器访问：http://localhost:1880/ui
   - 或点击右上角菜单 → `Dashboard`

2. **界面布局**
   - **极坐标雷达图**（上半部分）：半圆形雷达图，实时显示障碍物分布
     - 鼠标悬停 → 显示角度和距离
     - 鼠标点击 → 高亮选中该点（黄色）
   - **距离仪表盘**：径向表盘显示当前距离，量程 0-250cm
   - **状态提示**：绿色="运行正常"，红色="障碍物过近!"或"模块被遮挡"
   - **记录浏览器**：自动保存每次完整扫描的记录，可查看探测点明细
   - **角度滑块**：拖动控制舵机角度（0-180°）
   - **模式按钮**：切换自动扫描 / 手动模式

## 五、记录浏览功能

每次扫描完成（0°→180°→0°），系统自动保存一条记录。

### 查看记录

1. 在 **记录浏览器** 区域，点击下拉框
2. 选择要查看的记录（格式：`#ID 时间 (点数)`）
3. 下方表格显示该记录中所有探测点的 **角度** 和 **距离**
4. 距离 < 30cm 的点标红显示，带 ⚠ 标记

### 极坐标图交互

- **悬停**：鼠标在雷达图上移动，显示该位置的角度和距离
- **点击**：点击一个数据点，高亮显示（黄色），底部信息区显示精确值
- **被遮挡**：模块被遮挡时，图下方提示"模块被遮挡"
- **扫描线**：黄色虚线表示当前舵机指向角度

## 六、串口控制

### 通过Node-RED控制

1. **设置角度**
   - 拖动角度控制滑块 → 自动发送 `{"angle":90}` 到串口

2. **切换模式**
   - 点击"自动扫描"按钮 → 发送 `{"mode":"auto"}`
   - 点击"手动模式"按钮 → 发送 `{"mode":"manual"}`

### 通过串口助手发送命令

| 命令 | 功能 |
|------|------|
| `A` 或 `a` | 切换自动扫描模式 |
| `M` 或 `m` | 切换手动模式 |
| `D` 或 `d` | 切换OLED显示模式 |
| `{"angle":90}` | 设置舵机角度 |
| `{"mode":"auto"}` | 自动模式 |
| `{"mode":"manual"}` | 手动模式 |

## 七、数据格式

STM32 上行数据格式（每 ~200ms 发送一次）：

```json
{"angle":45,"distance":20.5,"warning":0,"blocked":0,"display_mode":1}
```

| 字段 | 说明 |
|------|------|
| `angle` | 当前角度 0-180 |
| `distance` | 距离 (cm) |
| `warning` | 0=正常, 1=障碍物过近 |
| `blocked` | 0=正常, 1=模块被遮挡 |
| `display_mode` | 0=文本, 1=雷达图 |

## 八、故障排查

### 问题1：串口连接失败

**原因**：
- COM口被占用
- 串口驱动未安装
- 波特率不匹配

**解决**：
- 检查设备管理器中的COM口
- 关闭其他串口软件（XCOM、SSCOM等）
- 重新安装USB转串口驱动

### 问题2：Dashboard无法访问

**原因**：
- node-red-dashboard未安装
- 端口被占用

**解决**：
- 在Node-RED目录下执行：`npm install node-red-dashboard`
- 重启Node-RED
- 检查1880端口是否被占用

### 问题3：数据不更新

**原因**：
- STM32未发送数据
- 串口连接问题
- JSON解析失败

**解决**：
- 检查STM32是否正常运行
- 检查串口连接
- 查看Node-RED调试信息（右侧 Debug 面板）

### 问题4：极坐标图不显示

**原因**：
- 浏览器不兼容
- 初始数据为空

**解决**：
- 使用Chrome或Edge浏览器
- 等待几秒直到第一个扫描数据到达

## 九、调试技巧

### 查看串口数据

1. **添加debug节点**
   - 在 `nr_json` 节点后连接一个 `debug` 节点
   - 部署后查看接收到的JSON数据

2. **测试串口收发**
   - 单独向STM32发送 `{"angle":90}` 测试舵机
   - 发送 `{"mode":"auto"}` 测试自动扫描

## 十、常用快捷键

- `Ctrl+S`: 保存流程
- `Ctrl+Z`: 撤销
- `Ctrl+Y`: 重做
- `Del`: 删除选中节点
- `Ctrl+C / Ctrl+V`: 复制粘贴
- `Ctrl+A`: 全选
- `Ctrl+D`: 部署
